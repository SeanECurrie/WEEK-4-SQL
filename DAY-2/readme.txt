1. ERD (Entity Relationship Diagram)
-- Illustrates how “entities” such as people, objects or concepts relate to each other within a database system.

2. Cardinality
-- Defines the numerical attributes of the relationship between two entities or entity sets.

3. Entities/Tables
-- 1 Primary Key (Unique Attribute), Attributes describing the table, Type of Attribute
-- Can also have Potential Foreign Keys

4. Primary Key Rules
-- Unique
-- Can never change
-- Never null

5. Foreign Key Rules
-- A Foreign Key is the same as a Primary Key, but just located in a Foreign place.
-- Foreign Keys don't have to be Unique. They can be repeated in a table.
-- There can be multiple Foreign Keys in one entity.

6. DQL, DDL, DML
-- DQL (Data Query Language)
-- Query data from database.

-- DDL (Data Definition Language)
-- Creating our tables/entities in our database.
-- When creating tables, make sure you create the tables that are not reliant on over tables AKA don't have foreign keys.

-- DML (Data Manipulation Language)
-- Inserting values into our database
